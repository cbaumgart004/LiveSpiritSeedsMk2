The `layered` and `refined` UX styles shipped (PR #6/#7) but the owner's verdict is that they need
serious revision — right now they look like the original page with poorer fonts and content stripped,
not a distinct experience. Watercolor (original) stays the default; this is about making the
alternates actually worth choosing.

Specific problems called out:
- **Media is missing.** Refined removes the watercolor `::before` wash entirely (`display:none`),
  leaving flat cream bands that feel empty/stripped. The styles subtract imagery instead of adding
  visual richness. Alternates need their own media treatment, not less.
- **Parallax isn't noticeable.** The `layered` depth effect (scroll-driven `background-position`
  drift 32%→68%) is imperceptible, and relies on CSS `animation-timeline: view()` which many browsers
  (Safari <26, Firefox) don't run at all — so for most visitors there's no parallax and no layering
  motion. Needs a real, visible parallax/layering approach with a working cross-browser story
  (likely a JS/IntersectionObserver + transform fallback, not CSS-scroll-only).
- **Overall shallowness.** Differentiation is currently ~just a display-font swap + token tweaks.
  The alternates need genuine structural/layout/media/motion changes to feel like different
  experiences, per the original "UX not UI" intent.

Scope notes / where things live:
- Styles: `src/styles/ui-styles.css` (`body.style-layered`, `body.style-refined`).
- Fonts chosen: Layered = Bricolage Grotesque, Refined = Marcellus (keep unless revisited).
- Preview to iterate: `/?preview` on the live site, or Site Settings → "Open live preview".
- See DESIGN.md §6 (UX styles / Preview mode) and ADR 0001 (CSS primitives).
